## 2025-01-15 - Event Listener Thrashing in React Scroll Handlers
**Learning:** In `Nav.jsx`, storing `lastScrollY` in React state (`useState`) and using it as a dependency in a `useCallback` for a `scroll` event handler caused massive performance issues. Every single pixel of scroll triggered a state update (re-render) AND recreated the `handleScroll` function. This recreation caused the `useEffect` to tear down and re-attach the `window.addEventListener('scroll')` on EVERY scroll event, leading to event listener thrashing and choppy scrolling.
**Action:** Always use `useRef` to store mutable values like scroll position that don't need to trigger a re-render. Additionally, add a check inside the handler to only call `setState` (like `setShowNav`) when the state actually needs to transition (e.g., `if (shouldHide && showNav)`), rather than updating state to the same value on every scroll event.
